Heydon's calculation, plus create separation from edge of viewport

This page demonstrates Heydon's calculation, plus adds padding that keeps the text away from the edge of the viewport at small screen sizes.

Holy crap this text is big.

I see conflicting advice and practices about how to accomplish keeping text away from the edge of the viewport. Do I really need a div to do this? Why can't I set this on body?

Early in the week I figured that this is one particular case where use of pixels is okay, although in the end I updated this page to switch to rem. And in theory I could add this as a media query because it's a concern only with small screens, but I assigned it to the body because the simplicity of doing so seems to outweigh the need to apply the margin only at small screen sizes.

html {
font-size: calc(1em + 1vw);
}

body {
padding: 0 .5rem;
}

No base font size or font family is specified.
