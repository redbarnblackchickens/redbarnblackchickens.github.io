From the clamp() spec, setting width

Responsive typography is a swamp. There is the issue of specifying font size and line height, and then the issue of addressing width.

In an attempt to carve out a piece of sanity, I looked to the clamp spec for help setting the width of an html page. Here's an example from the spec., which is used by the css on this page:

body { margin: 0 auto; width: min(1000px, calc(70% + 100px)); }

Line length should be around 60 to 75 characters, and obviously line length of this page is too long. About 2.5 alphabets is another way to gauge it.
