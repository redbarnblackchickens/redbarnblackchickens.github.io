Sampler #7: Yet another approach

On web.devin in fall 2019, Una Kravets has an article that proposes min(), max(), and clamp for fluid typograpy. That's the approach I know makes sense. In general she uses unitless values, which is good. And I see she also is aware of accessibility issues, such as this one: "Limiting how large text can get with max() or clamp() can cause a WCAG failure under 1.4.4 Resize text (AA), because a user may be unable to scale the text to 200% of its original size."

The css on this page, which is below, uses her general approach, esp. for clamp(). I'm struggling with setting page width.

body { margin: 1rem auto; width: 80%; }

h1 {
font-size: clamp(1.8rem, 2.5vw, 2.8rem);
}

p {
line-height: 1.6;
font-size: clamp(1rem, 1.3vw, 3rem);
width: clamp(25ch, 80%, 60ch);
}

code {
line-height: 1.2;
font-size: clamp(.9rem, 1.4vw, 1.6rem);
}
